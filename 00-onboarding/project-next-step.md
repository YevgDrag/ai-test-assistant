# Project Next Step

## Далі робимо

1. **Підготуємо приклад вхідних даних.**
   - Візьмемо одну реальну Jira issue та відповідну Confluence-сторінку.
   - Витягнемо з них: summary, description, acceptance criteria, повʼязані вимоги.

2. **Побудуємо простий prototpe API.**
   - Створимо невеликий Python-сервер, який приймає цей текст і повертає markdown тест-план.
   - На першому етапі без write-back у Jira.

3. **Зробимо prompt-driven генерацію.**
   - Напишемо шаблон prompt, який відповідає твоєму MVP:
     - перелік перевірок,
     - навіщо ці перевірки,
     - як їх виконати,
     - інструменти,
     - підтвердження QA.

## Чому це корисно

- Ти отримаєш швидкий робочий artefact, який можна показати колезі.
- Цей прототип стане основою для подальшої інтеграції з Jira.
- Ми збережемо просту архітектуру і не будемо заганяти себе у складні рішення на старті.
