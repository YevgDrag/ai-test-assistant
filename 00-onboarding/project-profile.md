# Project Profile

## Setup Summary

- **Мета:** створити AI-асистента для автоматизації QA-процесів, який аналізує вимоги з Jira-issue та Confluence, пропонує перелік можливих перевірок, отримує підтвердження від QA і генерує структурований тест-план у вигляді пунктів.
- **Роль:** QA інженер.
- **Команда:** проектний менеджер / бізнес-аналітик / продакт-оунер, дизайнер, фронтенд-розробник, бекенд-розробник, QA.
- **Перший кейс:** аналіз конкретної Jira-issue і пов'язаної сторінки в Confluence для генерації тест-плану.
- **MVP:** асистент приймає вхідні дані (текст вимог), аналізує їх і повертає структурований тест-план без автоматичного write-back у Jira на першому етапі.
- **Додатково:** якщо в задачі є фронт-енд зміни, асистент має питати про наявний дизайн або Figma-лінк.

## Повідомлення для колеги

1. Ми будуємо AI-асистента, який не просто пише тест-план, а пропонує перелік важливих перевірок, пояснює чому вони потрібні і якими інструментами їх можна виконати.
2. Перший запуск — це робочий прототип, який повертає результат у вигляді markdown-структури; думка QA важлива перед тим, як тест-план буде остаточно сформований.
3. Ми почнемо з MVP без автоматичного запису у Jira, а write-back додамо пізніше.
4. Безпека: всі дані з Jira/Confluence мають оброблятися акуратно і так, щоб конфіденційні фрагменти не потрапляли в загальний код або публічні артефакти.
