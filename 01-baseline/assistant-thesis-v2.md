# AI QA Assistant Baseline — v2

## Мета

Створити окремий baseline, який зберігає вихідну бізнес-ідею та MVP без зміни першого документу. Цей файл фокусується на найпростішому прототипі, який приймає опис вимог і повертає структурований тест-план.

## Опис MVP

- Вхідні дані: текст Jira вимоги, acceptance criteria, опціональні нотатки з Confluence.
- Вихідні дані: структурований тест-план у форматі markdown, перелік перевірок, та питання для підтвердження QA.
- Підхід: локальний FastAPI сервіс як інтерфейс MVP.
- Обмеження: на першому етапі не виконується автоматичний write-back у Jira.

## Основні компоненти

1. `06-implementation/app.py`
   - Простий API з POST `/generate-test-plan`.
   - Валідація запиту через `pydantic`.
   - Генерація тест-плану з базовою логікою для frontend та design контексту.

2. `06-implementation/prompt_template.md`
   - Шаблон для майбутнього LLM-промпту.
   - Визначає структуру вхідних даних та очікуваний результат.

3. `06-implementation/README.md`
   - Інструкція для запуску прототипу.
   - Приклад запиту `curl`.

4. `06-implementation/requirements.txt`
   - Залежності: `fastapi`, `uvicorn`, `pydantic`.

## MVP логіка

- Аналізує базові параметри вимоги.
- Формує перелік читабельних перевірок.
- Додає пояснення, чому кожна перевірка важлива.
- Підтримує просту адаптацію до дизайну й фронтенд-змін.

## Подальші кроки

- Додати інтеграцію з LLM, щоб замінити статичну генерацію на генерацію на основі промпту.
- Створити unit-тести для API.
- Додати конфігурацію для запуску у контейнері або сервері.
- Інтегрувати write-back у Jira/Confluence як наступний етап.
