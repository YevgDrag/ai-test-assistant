echo "# ai-test-assistant" >> README.md
git init
git add README.md
git commit -m "first commit"

git remote add origin https://github.com/YevgDrag/ai-test-assistant.git
git push -u origin main

------------------------------------------------------------------------------
Твій щоденний робочий цикл:
Після того, як ти відредагував файли в папці C:\Projects\ai-test-assistant, виконуй ці три кроки в терміналі:

cd C:\Projects\ai-test-assistant

git add .

git commit -m "Короткий опис того, що я зробив"

git push

================================================================================
Пояснення кожної команди:
git add . — це команда "додати все". Крапка (.) означає, що ти хочеш додати в індекс для відправки всі змінені, створені або видалені файли в поточній папці. Ти кажеш Git: "Ось ці зміни я хочу підготувати до збереження".

git commit -m "опис" — це "знімок" стану твого проєкту. Ти створюєш запис в історії Git. Параметр -m дозволяє написати коротке пояснення прямо в команді. Це критично важливо, щоб через місяць ти міг відкрити історію і зрозуміти, що саме ти робив у кожному коміті.

git push — це команда, яка відправляє (завантажує) твої локальні коміти на сервер GitHub. Після цього твої зміни стають видимими в репозиторії онлайн.

Чому не треба використовувати решту команд щоразу:
git init — ініціалізує репозиторій. Це робиться один раз у житті папки.

git branch -M main — перейменовує гілку. Робиться один раз.

git remote add origin ... — "знайомить" твою локальну папку з конкретним посиланням на GitHub. Робиться один раз.

git push -u origin main — в перший раз параметр -u (upstream) запам’ятовує, що твоя локальна гілка main має завжди відправлятися в origin на GitHub. Тому в майбутньому тобі достатньо просто писати коротке git push.

echo "# ai-test-assistant" >> README.md — це команда для створення або оновлення файлу README. Це не Git-команда, а команда операційної системи.

Моя порада для твого перфекціонізму:
Якщо ти забув, чи вніс зміни, або що саме змінив, завжди перед git add . виконуй команду git status. Вона покаже тобі червоним кольором усі змінені файли. Це дуже зручно, щоб випадково не "запушити" те, що ти ще не планував додавати.
