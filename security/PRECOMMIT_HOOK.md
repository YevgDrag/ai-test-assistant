# Pre-commit hook для сканера секретів

Цей документ пояснює, що таке `pre-commit` hook і як він працює з `scripts/check_secrets.py`.

## Що таке pre-commit hook?

`pre-commit` — це інструмент, який автоматично виконує перевірки перед тим, як створити коміт. Коли ви запускаєте `git commit`, hook перевіряє код і може зупинити коміт, якщо знайдено проблеми.

## Що я додав

- Файл `c:\Projects\ai-test-assistant\.pre-commit-config.yaml` з конфігурацією hook.
- Hook називається `check-secrets`.
- Він запускає скрипт `python scripts/check_secrets.py` перед кожним комітом.

## Як це працює

1. Ви виконуєте `git add <files>`.
2. Ви виконуєте `git commit`.
3. `pre-commit` читає `.pre-commit-config.yaml`.
4. Він виконує hook `check-secrets`.
5. Hook виконує скрипт `scripts/check_secrets.py`.

### Якщо скрипт повертає `0`
- Значить сканер не знайшов очевидних потенційних секретів.
- `git commit` продовжується.

### Якщо скрипт повертає `1`
- Значить сканер знайшов потенційні секрети.
- `git commit` блокується.
- Ви побачите список файлів і рядків, де знайдено потенційні секрети.

### Чому могла з'явитися помилка `InvalidConfigError`
- У старих версіях `pre-commit` тип `md` не розпізнається як допустимий тип файлу.
- Тоді конфігурація `.pre-commit-config.yaml` вважається некоректною, навіть якщо сама команда `python scripts/check_secrets.py` працює.
- Рішення: оновити `pre-commit` або використовувати `language: system` для локального hook.

### Що означає `language: system`
- `language: system` змушує `pre-commit` виконувати команду без додаткової внутрішньої обробки.
- Це зручно для простих локальних скриптів, які вже запускаються командою `python ...`.
- У цьому випадку ми використовуємо `language: system`, щоб уникнути несумісності по типах і версіях `pre-commit`.

## Як встановити `pre-commit`

У Python-середовищі виконайте:

```bash
pip install pre-commit
```

Потім встановіть hook для репозиторію:

```bash
pre-commit install
```

Після цього кожен раз перед комітом буде запускатися сканер.

## Як відключити тимчасово

Якщо ви впевнені, що потрібно пропустити перевірку на один коміт:

```bash
git commit --no-verify
```

## Додаткові переваги

- Ви не забуваєте запускати перевірку вручну.
- Непідготовлені секрети блокуються до моменту коміту.
- Це корисно для команди, щоб уникнути випадкової публікації ключів і паролів.

## Що робити при знаходженні проблем

1. Відкрийте файл і перевірте знахідку.
2. Якщо це дійсно секрет, видаліть його з файлу, переведіть у змінну середовища або секретне сховище.
3. Запустіть `python scripts/check_secrets.py` ще раз.
4. Якщо скрипт більше не знаходить проблем — повторіть коміт.
